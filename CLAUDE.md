## What This App Does

A run planning web app where users draw paths on a map (UK/Ireland region only: lat 50–60, lon -10 to 5) and view elevation profiles along those paths. Includes user authentication with JWT + refresh tokens.

## Commands

### Frontend (`client/`)
```bash
npm run dev       # Start Vite dev server on http://localhost:3000
npm run build     # Production build + TypeScript check
npm test          # Run Vitest unit tests
npm run check     # Biome lint + format check
npm run format    # Auto-format with Biome
```

### Backend (`LogInPage/`)
```bash
dotnet run                              # Start API on https://localhost:7204
dotnet ef migrations add <Name>         # Add EF Core migration
dotnet ef database update               # Apply migrations
```

### Infrastructure
```bash
docker compose up -d                    # Start PostgreSQL (5432) + Open-Elevation (80)
docker compose -f docker-compose-full.yml up  # Full stack including API container
```

## Architecture

**Frontend** (`client/src/`): React 19 + TypeScript, Vite, TanStack Router (file-based routing), Tailwind CSS 4, MUI v7. Routes live in `src/routes/` — do not manually edit `routeTree.gen.ts` (auto-generated by TanStack Router plugin). Path alias `@/` maps to `src/`.

**Backend** (`LogInPage/`): ASP.NET Core minimal API on .NET 10. Endpoints defined in `Controllers/UserController.cs`. DI wiring in `Extensions/ServiceCollectionExtensions.cs`. Database seeded/migrated on startup via `Extensions/WebApplicationExtensions.cs`.

**Auth flow**: Login returns a short-lived JWT access token (1 min, stored in cookie) + long-lived refresh token (1 day, HttpOnly cookie). `TokenCleanupService` runs every 60 min to prune expired refresh tokens from the DB.

**Elevation data**: Open-Elevation API (Docker container) backed by local SRTM GeoTIFF tiles in `elevation-data/`. The frontend calls this directly via `VITE_OPEN_ELEVATION_API_URL_BASE`.

**Database**: PostgreSQL in Docker. Uses ASP.NET Core Identity (schema: `identity`) extended with `Name` and `NotificationsEnabled` on `ApplicationUser`. Refresh tokens stored in a separate `RefreshTokens` table.
